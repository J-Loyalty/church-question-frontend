# Quiz Frontend - 프로젝트 정의서

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | quiz-frontend |
| 목적 | 오프라인/온라인 퀴즈 문제풀이 및 채점 시스템 |
| 기술 스택 | React 19, Vite 8, React Router 7 |
| 반응형 | PC (960px) / 모바일 (480px 이하) 대응 |
| 백엔드 연동 | REST API (프록시: `/api` → `http://localhost:8080`) |

---

## 2. 화면 흐름

```
[시작화면] → [문제풀이] → [완료화면] → [랭킹]
    │                                      │
    └──────────── 랭킹 링크 ───────────────┘
```

### 2.1 시작화면 (`/` - ready phase)
- 퀴즈 타이틀, 총 문제 수 표시
- **이름 입력** (필수) → 시작하기 버튼 활성화
- 시작하기 클릭 시 **시작 시간 기록**
- 하단에 랭킹 페이지 링크

### 2.2 문제풀이 (`/` - quiz phase)
- 상단 진행률 바 + 문제 번호 카운터
- 문제 카드에 질문 표시

#### 객관식 (type === 2)
- 보기 텍스트 길이에 따라 레이아웃 자동 결정
  - 모든 보기 10자 이하 → **가로 한줄** 배치
  - 하나라도 10자 초과 → **세로** 배치
- 보기 선택 시 **자동으로 다음 문제**로 이동 (200ms 딜레이)
- 선택 하이라이트 없음 (채점 힌트 방지)

#### 주관식 (type !== 2)
- "답변을 말해주세요" 안내 텍스트만 표시
- 문제 푸는 사람이 **구두로 답변**
- 채점자가 **화면 영역 클릭**으로 채점:
  - **오른쪽 클릭** → 정답 처리
  - **왼쪽 클릭** → 오답 처리
- 채점 UI가 화면에 보이지 않아 문제 푸는 사람이 인지 불가

#### 네비게이션
- 이전/다음 버튼으로 문제 이동 가능
- 마지막 문제에서는 제출 버튼 표시

### 2.3 완료화면 (`/` - done phase)
- "🎉 수고하셨습니다!" 메시지
- 이름, 소요시간 표시
- 랭킹 보기 / 다시 풀기 버튼

### 2.4 랭킹 페이지 (`/ranking`)
- 서버에서 랭킹 데이터 조회
- 카드 형태로 순위, 이름, 점수, 소요시간 표시
- 1~3위 색상 하이라이트 (금/은/동)
- **각 항목 클릭 시 아코디언으로 문제별 상세 결과 펼침**
  - 문제, 사용자 답변, 정답 여부 표시
  - 오답인 경우 정답도 함께 표시

### 2.5 정답 확인 페이지 (`/answers`)
- 전체 문제 목록을 테이블로 표시 (번호, 문제, 유형, 정답)
- 메인 화면에서 직접 링크 없음 (URL 직접 접근용)

## 4. API 명세

### 4.1 문제 조회
```
GET /api/quiz
```
**Response:**
```json
[
  {
    "description": "문제 내용",
    "type": 2,
    "correctAnswer": "정답",
    "answers": [
      { "num": 1, "description": "보기1" },
      { "num": 2, "description": "보기2" }
    ]
  }
]
```
- `type: 2` → 객관식 (answers 배열 포함)
- `type: 1` (또는 그 외) → 주관식

### 4.2 점수 저장
```
POST /api/quiz/score
Content-Type: application/json
```
**Request Body:**
```json
{
  "name": "홍길동",
  "score": 4,
  "total": 5,
  "elapsed": 127,
  "details": [
    {
      "question": "문제 내용",
      "userAnswer": "사용자 답변",
      "correctAnswer": "정답",
      "correct": true
    }
  ]
}
```
| 필드 | 타입 | 설명 |
|---|---|---|
| name | string | 문제 푸는 사람 이름 |
| score | number | 맞춘 문제 수 |
| total | number | 전체 문제 수 |
| elapsed | number | 소요시간 (초) |
| details | array | 문제별 상세 결과 |

### 4.3 랭킹 조회
```
GET /api/quiz/ranking
```
**Response:**
```json
[
  {
    "name": "홍길동",
    "score": 4,
    "total": 5,
    "elapsed": 95,
    "details": [
      {
        "question": "문제 내용",
        "userAnswer": "답변",
        "correctAnswer": "정답",
        "correct": true
      }
    ]
  }
]
```
- 점수 높은 순, 동점 시 소요시간 짧은 순 정렬
- `details` 포함하여 랭킹에서 문제별 결과 확인 가능

## 5. 주요 설계 결정

| 항목 | 결정 | 이유 |
|---|---|---|
| 주관식 채점 방식 | 투명 좌/우 영역 클릭 | 문제 푸는 사람에게 채점 UI 노출 방지 |
| 객관식 자동 넘김 | 선택 후 200ms 딜레이 | 빠른 진행 + 클릭 피드백 확보 |
| 보기 레이아웃 | 텍스트 길이 기반 자동 전환 | 짧은 보기는 가로, 긴 보기는 세로로 가독성 확보 |
| 결과 확인 | 랭킹 페이지에 통합 | 완료 후 바로 결과 노출 방지, 랭킹에서 상세 확인 |
| 소요시간 측정 | 시작 클릭 ~ 제출 시점 | 클라이언트 기준 측정 |
| 마지막 문제 답 처리 | goNext(val)로 직접 전달 | setState 비동기 문제 해결 |

## 7. 빌드 및 실행

```bash
# 의존성 설치
npm install

# 개발 서버 (http://localhost:5173)
npm run dev

# 프로덕션 빌드
npm run build

# 빌드 결과 미리보기
npm run preview
```

### 환경 요구사항
- Node.js 18+
- 백엔드 서버: `http://localhost:8080` (API 프록시 대상)
